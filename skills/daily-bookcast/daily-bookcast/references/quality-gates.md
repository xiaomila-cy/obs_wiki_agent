# 交付前质量门槛

## 一票否决项

出现任意一项都不能交付：

- 从收到书名到最终正文的墙钟时间超过 480 秒。
- 书名、作者、版本或体裁识别错误。
- 虚构核心事实、情节、结局、数据、引语或作者观点。
- 把决定高潮或结论的角色推测、竞争性解释或评论者判断写成文本确认的事实，从而改变核心责任或因果链。
- 在关键主张中漏掉否定、交换主客体、写反增减方向或扩大适用范围，导致研究、金融、政策或医学含义发生实质反转。
- 把只适用于特定样本、人群或技术层级的判断无边界扩展为普遍因果或当前共识；涉及胚胎、生殖系或可遗传干预时尤其如此。
- 把有争议的跨层桥接写成无条件事实，例如仅凭桥两端分别成立，就断言科学还原、普遍因果、正式身份、完整动机或组织隶属已经得到证明。
- 历史或传记的关键节点相对顺序错误，或者主题重组回到更早节点却使用前进型时间转场，导致事实先后或因果被实质改写。
- 严重遗漏决定全书含义的核心论点、关键因果或主要结局。
- 少于 6900 或多于 10600 个非空白字符，且没有用户要求或题材原因。
- 出现 Markdown 标题、代码块、项目符号、表格、网址、脚注、引用编号、Emoji 或装饰分隔线。
- 出现大段复制、明显套用其他书的内容、TODO、N 倍、待补充等占位符。
- 冒充主播、作者熟人、课程主理人或亲历者。
- 正文之外附加研究说明、评分、来源列表、道歉或创作过程。

## 证据充分性门槛

- 目录、书目页和章节名只能证明结构，不能单独证明概念解释、论证机制或作者结论。
- 每个主体部分的核心概念或关键因果至少绑定一项概念级证据；没有证据的具体解释必须删除或降格。
- 书外应用必须明确表现为沿着机制所作的延伸。无法区分原书主张与写作者推论时，事实准确性不得达到交付线。
- 事实底稿已经为四到六项最高风险内容建立主张卡，逐项记录主体、极性与方向、范围、证据层级、时间标签和允许使用的确定程度。每块落盘前均对照相关卡片，不能把最后五秒的粗略扫读当作语义核验。
- 任何决定主线的跨层因果、身份或范围连接已经作为桥接主张占用一张卡，并分别核过前件、后件和连接本身。两端证据不能替代桥接证据；连接只能得到作者路线或竞争性解释支持时，正文必须保留归属和争议。
- 小说中的文本明示、角色推测与讲稿解释已经分层；创作、出版与故事年代已经分栏。非虚构中的书内术语、作者其他作品术语和写作者概括已经分层。
- 小说必须完成关键阶段的能力与状态一致性检查，不能把人物早期的识字情况、伤病、已知信息、关系或其他限制错误延续到后期。任何改变关键因果的状态错误都会使事实准确性不通过。
- 经验研究型非虚构必须实际写入至少一项具名研究方法及结果、两个承担不同论证动作的原书案例和一个研究边界；哲学或纯理论作品必须有具名思想实验、文本论证或对话场景。缺项时，核心内容覆盖与信息密度均不得达到交付线。
- 历史传记必须先写清直接导火索，并实际写入至少三个来自不同章节的“所知、选择、代价、后果”节点；还必须为准备入稿的节点建立时间节点链，核对事实顺序、正文顺序和相邻时间边。主题重组可以回叙，但必须明确标记；顺序未定时不得使用前进型转场。具体人物、事件与选择原则上占主体约一半。只剩制度、信息、流程、资源或系统等抽象评论时，题材结构与核心覆盖不得达到交付线。
- 经验研究型作品的研究现场、原书案例及解释原则上占主体约三成五到五成。三个生产块各有至少一个已核验具体材料；连续两段抽象解释后仍没有材料落地时，叙事吸引力与信息密度不得达到 4 分。
- 研究阶段在前 40 秒内完成，第 50 秒锁定结构并开始第一块；第一、二块分别在约第 210、325 秒运行软进度检查。中间迟到或字符不足只产生 `warning` 或 `critical`，不永久作废；落后时立即删去次级材料并合并剩余主线。第三块正常在第 440 秒前落盘，第 440 至 455 秒只允许已在运行的末块返回或修复已知小缺口，第 455 秒正文绝对封口；`finish` 必须在第 475 秒前启动，总耗时不得超过 480 秒。
- 已在开始时建立绝对截止时间，并在每批网络工具返回、每个主体部分完成后检查剩余时间；不能依赖主线程提醒。
- 正文提纲已经按开场、主体部分和结尾分配约 7000 至 7150 字，再切成三个有界生产块；第一块约 1650 至 1750 字，第二块新增约 2450 至 2600 字并累计约 4100 至 4350 字，第三块依据第二项机器计数，按 6900 字硬线的缺口增加 100 至 250 字估长余量，一次完成剩余主线、必要边界和约 450 至 600 字收束。前两块落盘后直接运行编号 `checkpoint`，正常路径不得在它前面额外运行 `measure` 或 `status`；第三块后直接运行 `finish`。最终块必须达到 6900 字硬线。禁止把整篇长稿放进单个不可中断的工具调用，也不要退回细碎的逐段写入。
- `checkpoint` 是软进度警报。出现 `warning` 或 `critical` 时不重试小缺口，直接删除无法核实的精确细节、次级案例、作者履历、书外应用和重复总结，同时保留中心因果链、题材最小证据载荷、必要边界与完整结尾。只有第 455 秒正文封口、第 475 秒终检启动门和第 480 秒总时限属于运行硬门。

## 八维评分

每项按 0 至 5 分自评。交付线是每项不低于 4.0，平均不低于 4.5，并且一票否决项为零。

### 一、事实准确性

- 5：所有核心事实可追溯，高风险细节已交叉核验，假说、争议与共识边界清楚。
- 4：核心内容准确，只有不影响理解的低风险细节证据较弱。
- 3：总体方向正确，但存在未核验数字、引语或过度确定的断言。
- 0 至 2：出现核心错误、混书、编造或关键矛盾。

### 二、核心内容覆盖

- 5：抓住全书真正的主问题、论证或故事因果链，关键材料与结论都在。
- 4：核心完整，少量重要但次级的内容被合理压缩。
- 3：像扩写的简介，观点有了，但论据、机制、结局或边界不足。
- 0 至 2：严重漏掉核心，或大量篇幅偏离原书。

### 三、题材结构适配

- 5：结构由这本书的内容自然长出，分部之间层层推进。
- 4：结构清晰且适配，少数转折略显常规。
- 3：能读懂，但明显按固定模板或原目录机械压缩。
- 0 至 2：顺序混乱、因果断裂或题材轨道选错。

### 四、口语与 TTS 适配

- 5：不用看文字也能听懂，句式有变化，称呼稳定，数字和术语都被说清。
- 4：整体顺口，只有少量长句、括号或书面表达。
- 3：文字上清楚，朗读时容易丢线索或喘不过气。
- 0 至 2：充满排版符号、列表、碎片、拗口句或指代不清。

### 五、叙事吸引力

- 5：开场迅速建立问题，故事、案例和解释交替推进，始终有真实的信息期待。
- 4：大部分段落有推动力，中段偶有平直但不会明显失去注意力。
- 3：信息正确却像讲义，或依赖空洞反问和夸张词维持热闹。
- 0 至 2：枯燥罗列、反复注水或情节难以跟随。

### 六、连贯与转场

- 5：听众始终知道当前问题，段与段、部分与部分有明确因果或疑问连接。
- 4：整体顺滑，少数段落与主线连接较弱。
- 3：局部内容不错，但像几篇笔记拼在一起。
- 0 至 2：跳跃、重复、前后矛盾或突然结束。

### 七、信息密度

- 5：几乎每段都在提供材料、解释关系或推进结论，具体而不拥挤。
- 4：密度稳定，少量背景或总结可以再收紧。
- 3：有明显履历堆砌、常识扩写、重复总结或术语拥堵。
- 0 至 2：大量空话，或大量细节没有解释作用。

### 八、语言风格一致性

- 5：亲切、清楚、有判断、有画面，保持长篇讲述节奏，同时语言原创。
- 4：大体符合，少数段落过书面、过随意或过于公式化。
- 3：像普通书评、百科摘要、营销稿或课堂提纲。
- 0 至 2：明显模仿具体原句、语体混乱或充满机器套话。

## 修订顺序

按以下顺序修改，避免为了听感掩盖事实问题：

1. 修正书籍识别、事实、因果和核心遗漏。
2. 重建主线和分部结构。
3. 补充真正有解释力的场景、案例、机制和边界。
4. 删除重复、履历堆砌、空洞反问和泛化感悟。
5. 拆长句、理顺指代、稳定称呼和转场。
6. 清理所有格式噪声，运行自动检查。
7. 确认墙钟耗时不超过 480 秒；超时不能以内容高分抵消。
8. 第三块正常在第 440 秒前落盘，第 455 秒正文绝对封口，随后只调用一次 `runtime_guard.py finish`，原子完成最终计数、格式检查和 `completed_at` 写入；开始阶段的 `start` 成功即完成脚本预检，不再运行 `finish --help`，也不能把最终计数、格式检查与计时拆成多个末尾往返。`finish` 必须在第 475 秒前启动，总耗时不得超过 480 秒；超时或格式失败必须留下 `passed: false` 的失败元数据。

不要根据总分平均掩盖短板。任一维度低于 4 分，继续修订。
